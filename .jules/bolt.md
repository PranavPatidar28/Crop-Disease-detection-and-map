## 2024-06-25 - Avoid new Date() for Sorting ISO 8601 Strings
**Learning:** Instantiating `new Date()` and calling `.getTime()` inside a `.sort()` callback creates massive overhead, particularly in loops or when trimming large local caches (like `live-reports.store.ts` or `notifications.store.ts`). Prisma and backend responses provide `createdAt` as ISO 8601 strings (e.g., `2024-01-01T12:00:00.000Z`), which are specifically designed to be sorted lexicographically as strings.
**Action:** When filtering or sorting lists of records by date on the frontend, always use direct lexicographical string comparison with basic operators (e.g., `b.createdAt > a.createdAt ? 1 : b.createdAt < a.createdAt ? -1 : 0`) instead of parsing to `Date` objects. Do not use `.localeCompare()` as it introduces Unicode collation overhead. This dramatically improves client-side performance.
